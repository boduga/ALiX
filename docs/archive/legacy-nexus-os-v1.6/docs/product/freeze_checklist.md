# Documentation Freeze Checklist

## Freeze Target

Freeze the split documentation pack before M0.9 coding expands beyond scaffolding.

## Required

- [ ] Slim PRD exists and is stakeholder-facing.
- [ ] Architecture specs extracted from old large PRD.
- [ ] M0.9 implementation plan approved.
- [ ] Model routing validation spike defined.
- [ ] Minimal M0.9 metrics defined.
- [ ] A2A remains gated and out of M0.9.
- [ ] No new product domain added to M0.9.
- [ ] Implementation scaffold created.
- [ ] Docs and schemas are versioned.

## Change Control

After freeze:

- PRD changes require explicit approval.
- M0.9 scope changes require a change request.
- Architecture details may evolve in supporting specs only.
- Coding starts from M0.9 plan, not from the full old PRD.

# M0.10-E: Report Artifacts and UX

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans.

**Goal:** Add `alix report list/show/open` commands and link reports ↔ graphs ↔ SOPs.

**Architecture:** Reads manifests from `.alix/reports/` directories. Updates `writeReportArtifacts()` to include graphId, sopId, topic, nodeResults. CLI reads manifests directly — no DB projection needed yet.

### Task 2: Add report CLI commands

**Files:** `src/cli.ts`

- [ ] Add `alix report list`:

Reads `.alix/reports/` directory. For each subdirectory, reads `run_manifest.json`. Sorts by `createdAt` descending. Prints table.

- [ ] Add `alix report show <reportId>`:

Reads `.alix/reports/<reportId>/run_manifest.json`. Prints full metadata + artifact file list.

- [ ] Add `alix report open <reportId>`:

Reads and prints `.alix/reports/<reportId>/final_report.md` to stdout.

- [ ] Add `alix report path <reportId>`:

Prints the absolute path to the report directory (for use with `cat`, `less`, etc.)

- [ ] Update `alix sop run` to print `Report: .alix/reports/<reportId>/` after execution.

- [ ] Build, commit `feat(cli): add alix report list/show/open/path commands`

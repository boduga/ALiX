# Migrate JavaScript Tooling to pnpm

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Migrate ALiX from npm to pnpm as the sole JavaScript package manager

**Architecture:** ALiX already has `packageManager` set to pnpm in `package.json` but the lockfile, CI, scripts, and docs all still reference npm. This plan completes the migration by generating `pnpm-lock.yaml`, adding `pnpm-workspace.yaml`, updating CI workflows, scripts, and documentation to use pnpm commands exclusively.

**Tech Stack:** Node.js 24, pnpm 11, GitHub Actions, TypeScript

## Global Constraints

- Use pnpm commands everywhere: `pnpm install`, `pnpm add`, `pnpm run`, `pnpm dlx`, `pnpm exec`
- `npm publish` is retained conservatively in the release workflow for this migration PR. pnpm 11 supports publishing natively, but switching the publish path is deferred to a separate cleanup PR to keep this migration focused on install/build/test determinism.
- One-off scripts outside the repo may use npm
- Do NOT remove `packageManager` field from `package.json` — it's already set to pnpm
- Keep `package-lock.json` removed after migration is validated
- Keep `node_modules` cleaned and regenerated via pnpm
- CI must use `--frozen-lockfile` for deterministic installs

## Current State

ALiX's JS tooling is in a partial migration state:
- `package.json` has `"packageManager": "pnpm@11.9.0+sha512..."` — pnpm is declared
- `package.json` has `"workspaces": ["packages/tool-repair"]` — workspace config
- But `package-lock.json` (npm) is the active lockfile (101KB)
- No `pnpm-lock.yaml` exists
- No `pnpm-workspace.yaml` exists (npm workspaces in package.json work differently than pnpm)
- `.npmrc` has npm-specific settings (`package-lock=true`)
- CI workflows (7 jobs in `ci.yml`, 1 in `publish.yml`) all use `npm ci`, `npm run`, and `cache: npm`
- `scripts/release-gate.sh` uses `npm run`, `npm pack`
- `docs/` references npm in quickstart, features, and release docs
- `packages/tool-repair` is an existing npm workspace

## Task 1: Generate pnpm lockfile and workspace config

**Files:**
- Create: `pnpm-lock.yaml`, `pnpm-workspace.yaml`
- Delete: `package-lock.json`
- Modify: `.npmrc`

- [ ] **Step 1: Generate pnpm lockfile from npm lockfile**

```bash
pnpm import
```

This reads `package-lock.json` and produces `pnpm-lock.yaml`.

- [ ] **Step 2: Create pnpm-workspace.yaml**

pnpm settings go in `pnpm-workspace.yaml`, not `.npmrc`. `.npmrc` is for registry/auth only.

```yaml
# pnpm-workspace.yaml
packages:
  - "packages/*"

# pnpm settings (not in .npmrc — pnpm ignores those there)
saveExact: true
strictPeerDependencies: false
autoInstallPeers: true
sharedWorkspaceLockfile: true
```

This matches the existing `"workspaces": ["packages/tool-repair"]` in `package.json`.

- [ ] **Step 3: Clean .npmrc**

Replace existing npm-specific `.npmrc` (which had `save-exact=true`, `min-release-age=2`, `package-lock=true`) with a minimal registry-only file:

```ini
# npm registry settings only.
# pnpm settings live in pnpm-workspace.yaml
package-lock=false
```

- [ ] **Step 4: Clean and reinstall**

```bash
rm -rf node_modules
rm package-lock.json
pnpm install
```

- [ ] **Step 5: Verify install**

```bash
ls node_modules/.pnpm 2>/dev/null && echo "pnpm store present"
pnpm list --depth=0 2>&1 | head -5
```

- [ ] **Step 6: Commit**

```bash
git add pnpm-lock.yaml pnpm-workspace.yaml .npmrc
git rm package-lock.json
git commit -m "chore: generate pnpm lockfile and workspace config"
```

- [ ] **Step 1: Update ci.yml — all 7 jobs**

In each job, replace:
```yaml

  with:
    node-version: "24"
    cache: npm

```
with:
```yaml
- uses: actions/setup-node@...

    node-version: "24"
```

The `corepack enable` step activates pnpm. The `--frozen-lockfile` flag makes CI fail if `pnpm-lock.yaml` needs updating (equivalent to `npm ci`).

- [ ] **Step 6: Add agent guardrail check:package-manager**

Create `scripts/check-package-manager.sh`:

```bash
#!/usr/bin/env bash
set -euo pipefail

echo "Checking package manager policy..."

if [ -f package-lock.json ]; then
  echo "❌ package-lock.json is not allowed. Use pnpm-lock.yaml."
  exit 1
fi

if [ -f yarn.lock ]; then
  echo "❌ yarn.lock is not allowed. Use pnpm-lock.yaml."
  exit 1
fi

if grep -R "npm ci\|npm install\|npm run\|npx " \
  .github/workflows scripts docs package.json \
  --include='*.yml' \
  --include='*.yaml' \
  --include='*.sh' \
  --include='*.md' \
  --include='package.json' \
  2>/dev/null | grep -v "npm publish" | grep -v "package-manager"; then
  echo "❌ Found disallowed npm/npx references."
  exit 1
fi

echo "✅ Package manager policy clean."
```

Make it executable:

```bash
chmod +x scripts/check-package-manager.sh
```

Add to `package.json` scripts:

```json
"check:package-manager": "bash scripts/check-package-manager.sh"
```

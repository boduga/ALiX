# P12.4 — Run Ledger Design Spec

**Date:** 2026-07-04
**Status:** Implementation-ready.

## Purpose

Persist the full governance decision trail for every autonomous run. P12.4 answers **"What proves it ran?"** It is the black box recorder for autonomous governance: it does not fly the plane, it proves what happened.

## Key Invariant

**Record evidence, don't decide anything.** P12.4 does not approve, deny, score, evaluate, execute, or merge. It accepts already-computed objects from P12.1, P12.2, and P12.3 and persists them.

## Core Types

```typescript
export type LedgerOutcome = "completed" | "failed" | "cancelled" | "denied";

export interface LedgerEntry {
  runId: string;
  issueId: string;
  policyResult: GovernancePolicyResult;
  riskScore: RiskScore;
  approvals: ApprovalGate[];
  filesChanged: string[];
  verificationResults: VerificationResult[];
  draftPrId?: string;
  outcome: LedgerOutcome;
  timestamp: string;
}
```

Where `GovernancePolicyResult` is from P12.1, `RiskScore` from P12.2, `ApprovalGate` from P12.3.

## Storage

JSONL (JSON Lines) format — append-only, auditable, easy to inspect.

```
.alix/governance/run-ledger.jsonl
```

Each line is a JSON-serialized `LedgerEntry`. New entries are appended. Listing reads the file and returns entries newest-first.

## Interface

```typescript
export interface RunLedgerStore {
  append(entry: LedgerEntry): Promise<void>;
  list(limit?: number): Promise<LedgerEntry[]>;
  get(runId: string): Promise<LedgerEntry | undefined>;
}
```

## Functions

```typescript
createLedgerEntry(input: CreateLedgerEntryInput, timestamp: string): LedgerEntry  // pure except for caller-supplied timestamp

listLedgerEntries(store: RunLedgerStore, limit?: number): Promise<LedgerEntry[]>   // delegates to store.list
getLedgerEntry(store: RunLedgerStore, runId: string): Promise<LedgerEntry | undefined>

```

## Validation Rules

| Field | Rule |
|-------|------|
| `runId` | Required, non-empty string |
| `issueId` | Required, non-empty string |
| `policyResult` | Required, must have `decision` field |
| `riskScore` | Required, must have `level` and `score` fields |
| `approvals` | Required, array |
| `filesChanged` | Required, array of strings |
| `verificationResults` | Required, array |
| `outcome` | Required, must be one of `completed | failed | cancelled | denied` |
| `timestamp` | Required, non-empty string |

## File Format

```
.alix/governance/run-ledger.jsonl
```

- Append-only: new entries are added at end of file
- Each line: `JSON.stringify(entry)`
- Malformed lines: skipped with a warning, not fatal
- Listing: read all lines, parse valid JSON, return newest-first (reverse order)

## CLI Shape

```bash
alix runs list [--limit N] [--json]
alix runs show <runId> [--json]
alix runs append --json '<entry>'    # dev-only, hidden in help
```

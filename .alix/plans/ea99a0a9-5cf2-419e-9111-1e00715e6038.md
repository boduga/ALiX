## Summary
Implement a robust directory listing utility module (`ls` equivalent) to programmatically retrieve, filter, and format file and directory metadata within the application.

## Changes

### 1. Create directory utility module
- **Action:** create
- **File:** `src/utils/fs_helper.py`
- **Description:** Create a new utility file containing a `list_directory` function. This function will mimic standard `ls` behavior (including options for showing hidden files, recursive listing, and long-listing formats like file size, permissions, and modification dates) using built-in system libraries (e.g., `pathlib` and `os`).

### 2. Register CLI or entrypoint command
- **Action:** modify
- **File:** `src/main.py`
- **Description:** Integrate the new listing utility into the main application entry point or CLI controller, allowing users to invoke the directory listing feature via a command-line interface or application argument.

### 3. Add unit tests
- **Action:** create
- **File:** `tests/test_fs_helper.py`
- **Description:** Implement unit tests to verify:
  - Accurate listing of a mock directory structure.
  - Correct behavior of flags (e.g., showing hidden files, recursive listings).
  - Proper error handling for non-existent directories or permission denied errors.
